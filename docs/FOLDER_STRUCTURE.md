
### `/src` Directory Structure

The `src` folder contains all React frontend code:

```
src/
├── main.jsx          # Entry point (imports App and mounts to DOM)
├── index.css         # Global CSS styles
├── App.jsx           # Root component with routes
│
└── pages/            # Page components
    ├── Dashboard.jsx       # Main weather dashboard
    ├── AlertsPage.jsx      # Alert management
    ├── MapsPage.jsx        # Map visualization
    ├── HistoryPage.jsx     # Historical data
    ├── ProfilePage.jsx     # User profile
    ├── SettingsPage.jsx    # User settings
    ├── SearchResults.jsx   # Search results
    ├── LoginPage.jsx       # Authentication
    └── LandingPage.jsx     # Home page
```

## File Organization Principles

### Frontend Organization (`src/`)

- **Modular Structure**: Each page is a standalone component
- **Route-Based**: `pages/` folder maps to application routes
- **CSS Co-location**: Styles next to components (in progress)
- **Single Entry Point**: `main.jsx` initializes the app

### Documentation Organization (`docs/`)

- **Getting Started**: `INSTALLATION.md` for new developers
- **API Reference**: `API_DOCUMENTATION.md` for backend endpoints
- **Architecture**: `ARCHITECTURE.md` for system design
- **Structure**: `FOLDER_STRUCTURE.md` (this file)

## Import Paths Examples

## Environment Files

### `.env.example`
Template file committed to GitHub showing all required variables.

### `.env` (local only, not committed)
Your actual API keys and secrets.

**Important**: Never commit `.env` file - it contains sensitive data!

## Dependency Locations

- **Frontend**: Listed in `package.json`
- **Backend**: Listed in `backend/requirements.txt`
